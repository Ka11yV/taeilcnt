# 태일씨엔티 공식 홈페이지 리뉴얼

태일씨엔티의 기업 신뢰도와 전문성을 효과적으로 전달하기 위해 구축한 정적 React/Vite 기반 공식 홈페이지 리뉴얼 프로젝트입니다.

## 프로젝트 개요

기존 홈페이지의 노후화된 UI를 개선하고, 외부 방문자·지원자·일반 사용자가 기업 정보를 빠르게 확인할 수 있도록 정보 구조와 사용자 경험을 재구성했습니다.

- 프로젝트명: 태일씨엔티 공식 홈페이지 리뉴얼
- 목표: 기업 소개, 사업 실적, ESG, 채용 정보를 신뢰감 있게 전달하는 반응형 웹사이트 구축
- 키워드: Trust & Professional
- 개발 방식: 정적 웹사이트

## 주요 특징

- 기업 홈페이지에 맞는 깔끔하고 안정적인 UI 구성
- 모바일 / 태블릿 / 데스크톱 반응형 대응
- 메뉴 구조 기반의 명확한 정보 설계
- 회사소개, 사업실적, 홍보센터, ESG경영, 인재채용 콘텐츠 통합 제공
- 이미지/영상 중심의 기업 소개 콘텐츠 구성
- 정적 호스팅 환경을 고려한 프론트엔드 구조

```text
회사소개
├─ 인사말

├─ 주 거래 시공사
├─ 업·면허/인증
└─ 찾아오시는 길

사업실적
├─ 공사수주 현황
├─ 건설시공능력
├─ 주택
├─ 업무시설
├─ 교육/의료
├─ 플랜트
├─ 초고층
├─ 판매시설
├─ 기타
└─ Project Map

홍보센터
├─ News
└─ 유튜브

ESG경영

인재채용
```

자세한 메뉴 기준은 `menu_structure.md`를 참고하세요.

## 기술 스택

- React 18
- Vite 5
- React Router DOM 6
- Lucide React
- JavaScript (ES Modules)

## 실행 방법

### 1) 의존성 설치

```bash
npm install
```

### 2) 개발 서버 실행

```bash
npm run dev
```

### 3) 빌드

```bash
npm run build
```

### 4) 빌드 결과 미리보기

```bash
npm run preview
```

## 폴더 구조

```text
.
├─ src/
│  ├─ assets/          # 이미지, 영상 등 프로젝트 리소스
│  ├─ data/            # 사이트 콘텐츠 데이터
│  ├─ App.jsx          # 메인 페이지/레이아웃 구성
│  ├─ main.jsx         # 앱 진입점
│  └─ styles.css       # 전역 스타일
├─ static/             # 원본 참고/제공 리소스
├─ reference/          # 기존 사이트 분석 및 참고 문서
├─ menu_structure.md   # 사이트 메뉴 구조 문서
├─ index.html
├─ vite.config.js
└─ package.json
```

## 콘텐츠 및 리소스 기준

- `/static` 폴더의 제공 리소스를 우선 활용
- 기존 태일씨엔티 홈페이지의 사진 및 문구를 참고하여 구성
- 정적 배포 환경을 고려해 외부 CDN 의존도를 최소화
- 상업적 이용이 가능한 리소스를 우선 사용

## 개발 원칙

- 정보 전달력을 우선하는 기업형 UI 구성
- 과도한 애니메이션보다 신뢰감 있는 인터랙션 사용
- 경로 문제를 줄이기 위해 정적 자산 관리에 유의
- 서버 사이드 로직 없이 클라이언트 중심으로 페이지 구성
- 제출 시 `node_modules`, 캐시 파일, `.env`, 임시 파일 제외

## 배포/제출 유의사항

- 정적 호스팅 환경에서 정상 동작하도록 구성해야 합니다.
- 이미지, 영상, 폰트 등 필요한 리소스는 프로젝트 내부에 포함하는 것을 권장합니다.
- 발표 및 제출 환경에서 경로 이슈가 없도록 상대 경로 기반 자산 관리가 중요합니다.
