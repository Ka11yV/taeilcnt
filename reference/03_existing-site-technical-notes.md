# 기존 사이트 기술 메모

## 1. 현재 구현 방식 요약

현재 사이트는 전형적인 **레거시 JSP + 정적 HTML 혼합 구조**다.

- 홈: `index.jsp`
- 목록형/검색형/다운로드형 페이지: `*.jsp`
- 소개형 일부 페이지: `*.html`
- 파일 다운로드: `filedown.jsp`
- 게시판형 상세: `order_view.jsp` 등

즉, 화면은 정적 페이지처럼 보이는 부분도 많지만 실제로는 서버 사이드 게시판/파일 시스템에 기대는 부분이 남아 있다.

---

## 2. URL/경로 특징

### 확인된 특징
- 루트 링크가 `/` 인 경우와 `/home/index.jsp` 인 경우가 혼재
- 상대경로와 절대성 경로가 섞여 있음
- `/home/file/upload/...` 형태의 업로드 경로 사용
- 일부 이미지/자료는 `intranet` 또는 `test` 하위 도메인에 존재

### 리뉴얼 시 의미
- 정적 호스팅 기준으로는 경로를 **일관된 상대경로 또는 명확한 base 전략**으로 재정리해야 함
- 외부/내부 도메인에 분산된 자산은 프로젝트 내부로 이관하는 것이 안전함

---

## 3. 인코딩/문서 선언 메모

공개 페이지를 수집한 결과, 사이트 전반은 **EUC-KR 기반 응답**이 중심이다.

다만 일부 게시판 페이지는 아래 문제가 보였다.

- HTTP 응답 헤더는 `euc-kr`
- 문서 내부 `meta charset` 는 `UTF-8`

즉, **인코딩 선언이 일관되지 않은 페이지가 존재**한다.

### 리뉴얼 시 권장
- 전체 문서를 **UTF-8로 통일**
- 파일명/메타/서버 응답 헤더 모두 일치시키기

---

## 4. 확인된 라이브러리/스크립트

홈 및 서브페이지에서 확인된 주요 의존성은 아래와 같다.

- jQuery 1.11.2 (Google CDN)
- Bootstrap (구버전 스타일 구조)
- Modernizr
- Vegas (메인 비주얼 배경 슬라이드)
- Swiper (프로젝트 슬라이더)
- matchHeight
- Sly
- Sky-forms (게시판/폼 UI 일부)
- classie / demo 스크립트류

### 해석
- 최신 프론트엔드 아키텍처라기보다 **플러그인 중심 조합형 구조**
- 유지보수와 자산 통합 측면에서 분산도가 높음

## 5. 외부 의존성 메모

### 공통 CDN/외부 로드
- Google Fonts
- Google Hosted jQuery
- html5shiv / respond.js CDN

### 외부 임베드/스크립트
- YouTube 임베드(홈 기업 소개 영상)
- Daum RoughMap 스크립트(찾아오시는 길)

### 내부 외부망/사내망 자산 의존
- `intranet.taeilcnt.co.kr` 이미지
- `test.taeilcnt.co.kr:8080` 이미지
- 일부 홍보자료/FAQ 이미지도 외부 내부망 주소 사용

### 시사점
- 현재 사이트는 “공개 사이트 단독 완결형”이 아니라 **사내 시스템/외부 CDN에 일부 의존**한다.
- 제출형 정적 사이트에서는 이 의존성을 제거하거나 명확히 대체해야 한다.

## 6. 페이지 유형별 구현 패턴

### 6.1 소개형 페이지
- 회사소개, 경영이념, 기술혁신 비전 등
- 비교적 단순한 텍스트/이미지 조합

### 6.2 이미지 의존형 페이지
- 기구 조직도
- 인사제도
- 주 거래 시공사
- 일부 홍보자료

### 6.3 게시판형 페이지
- 홍보센터 News
- 사회공헌
- 기술혁신 News
- 채용공고
- 채용FAQ
- ESG경영 일부

### 6.4 데이터/테이블형 페이지
- 건설시공능력
- 수주현황 일부

### 해석
리뉴얼 시 페이지 유형을 다음처럼 재설계하면 좋다.

- 소개형 → 콘텐츠 섹션형
- 이미지 의존형 → 데이터/카드/타임라인형
- 게시판형 → 카드형 리스트 + 필터/검색
- 데이터형 → 표 + 요약지표 + 강조 카드

---

## 7. 확인된 구조적 이슈

### 7.1 환경경영/안전경영/ESG 경계 모호
- `환경경영` 메뉴 URL에서 실제로는 안전보건 성격의 텍스트/placeholder가 보임
- ESG 관련 탭도 내용 분리가 충분하지 않음

### 7.2 빈 메뉴 존재
- 기술혁신 News: 0건
- 채용공고: 0건

### 7.3 이미지 기반 정보 손실
- 조직도/인사제도는 이미지에 정보가 묶여 있어 검색성과 접근성이 낮음

### 7.4 내부 링크 혼재
- 공개 홍보용 사이트에 내부 업무용 링크가 함께 노출됨

---

## 8. 정적 웹사이트 리뉴얼 관점 체크리스트

### 반드시 정리할 것
1. 모든 문서/데이터를 UTF-8 기반으로 통일
2. 상대경로 중심으로 재설계
3. 외부 CDN 및 내부망 이미지 자산 최소화 또는 내부 이관
4. 게시판형 콘텐츠를 정적 데이터(JSON/MD/로컬 이미지) 구조로 변환
5. 이미지에 갇힌 정보를 텍스트화

### 있으면 좋은 것
1. 프로젝트 리스트용 필터 UI
2. 연혁/인증/시공능력 시각화
3. 채용 FAQ/복리후생 카드화
4. ESG/사회공헌 콘텐츠의 통합 아카이빙
